# Симулятор свободного мира

Аудиоигра на `Swift` про свободное перемещение по квартире и двору.

Это не просто набор экранов, а живая звуковая сцена: игрок ходит по комнатам, находит предметы, слушает улицу, может выйти во двор, ориентироваться по звукам и взаимодействовать с миром.

## Что уже есть

- квартира с несколькими комнатами;
- предметы, которые можно находить и осматривать;
- подсказки о том, что находится рядом;
- озвучка действий и состояния мира;
- улица с машинами и дворовой атмосферой;
- отдельный мост управления для отладки и живых проверок;
- сценарии для быстрой проверки механик без долгого ручного прохождения.

## Что недавно улучшили

- большой файл `GameViewModel` был разложен на несколько понятных частей;
- стоящие машины на улице теперь можно найти и осмотреть как нормальные объекты;
- проезжающие машины не мешают осмотру и работают как фон;
- поведение машин стало мягче: заезд, остановка и уход ощущаются естественнее;
- добавлен более мягкий тормозной звук;
- в проект добавлен `MCP`, через который можно вызывать игровые сцены и быстрее искать баги.

## Планы

Егор продумал такую идею для будущего развития игры:

> В будущем машины нужны не только как уличный фон, но и как полноценная игровая механика. Игрок сможет подойти к припаркованной машине, сесть в неё и поехать до выбранной точки, например до магазина.
>
> Вождение должно работать по звуку: диктор подсказывает, куда держаться, а игрок ведёт машину так, чтобы основной звук оставался по центру. Если слишком сильно увести машину в сторону, можно врезаться. Это нужно будет отдельно продумать и настроить.
>
> В этой системе игрок не гонится за кем-то, а спокойно едет до нужной точки. После прибытия он выходит из машины и продолжает игру уже пешком.
>
> При этом важная идея мира в том, что до любой локации можно будет добраться и пешком. Для этого нужно продумать расположение мест, расстояния, координаты и связь между пешим перемещением и поездкой на машине.

## Как устроен проект

### Главное

- `Sources/Shared/GameViewModel.swift`
  Основа игрового состояния.

- `Sources/Shared/GameViewModel+Session.swift`
  Запуск игры, обработка команд, работа с отладочными сценами.

- `Sources/Shared/GameViewModel+Movement.swift`
  Ходьба по комнатам и по двору.

- `Sources/Shared/GameViewModel+Actions.swift`
  Действия игрока, осмотр, предметы, инвентарь.

- `Sources/Shared/GameViewModel+World.swift`
  Что находится рядом, что говорить игроку, как подхватываются объекты.

- `Sources/Shared/GameViewModel+Neighbor.swift`
  Всё, что связано с соседями.

- `Sources/Shared/GameViewModel+Audio.swift`
  Связь игры с озвучкой и звуком мира.

### Звук и улица

- `Sources/Shared/AudioCoordinator.swift`
  Общая работа со звуком.

- `Sources/Shared/SpeechCoordinator.swift`
  Озвучка текста.

- `Sources/Shared/StreetTrafficCoordinator.swift`
  Уличные машины: движение, парковка, звук, остановка, уезд.

### Мир

- `Sources/Shared/WorldCore/WorldModels.swift`
  Основные сущности мира и звуков.

- `Sources/Shared/WorldCore/WorldBuilder.swift`
  Сборка мира, комнат и двора.

- `Sources/Shared/Rooms`
  Отдельные комнаты и предметы.

### Отладка и проверка

- `Sources/MCPServer/main.swift`
  Сервер, через который можно управлять игрой извне.

- `Sources/Shared/LiveGameBridge.swift`
  Связь живой игры и внешнего управления.

- `Sources/Playtests/main.swift`
  Быстрые игровые проверки.

## Что собирается

В проекте есть несколько целей:

- `FreeWorldMac` — версия для macOS;
- `FreeWorldiOS` — версия для iPhone;
- `FreeWorldPlaytests` — проверки игровых сценариев;
- `FreeWorldMCP` — управление игрой извне;
- `FreeWorldVarispeedProbe` — отдельная маленькая проверка звука.

## Как запустить

Если есть `XcodeGen`, можно сначала пересобрать проект:

```bash
xcodegen generate
```

Сборка для macOS:

```bash
xcodebuild -project FreeWorldSimulator.xcodeproj -scheme FreeWorldMac build
```

Сборка `MCP`:

```bash
xcodebuild -project FreeWorldSimulator.xcodeproj -scheme FreeWorldMCP build
```

## Зачем этот проект

Идея проекта — сделать живой, понятный и интересный аудиомир, где пространство ощущается не описанием на экране, а звуком, предметами, расстоянием и реакцией мира на игрока.
